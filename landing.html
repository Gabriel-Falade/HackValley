The Problem

Gaming is inaccessible for millions of patients.

Hospital patients recovering from surgery, stroke, or injury spend weeks unable to use their hands. Gaming is one of the only forms of entertainment available during long stays — yet it remains completely out of reach. Existing solutions cost hundreds of dollars and require caregiver setup.

👁

Tobii Eye Trackers

Professional eye-tracking devices cost $150 to $3,000+ per unit. Not found in hospital rooms.

🪫

Adaptive Controllers

Require specific accessories, caregiver configuration, and a long setup time per patient.

😷

Voice-Only Solutions

Limited to simple commands with high latency — not suitable for fast-paced games.

💰

The Cost Barrier

Most hospitals cannot afford per-patient assistive hardware. Software-only is the only viable path.

How It Works

Your face is the controller.

FacePlay uses computer vision and a trained ML model to read your facial gestures in real time and convert them to keyboard inputs for any game.

01

Webcam Capture

OpenCV reads the camera at 60fps. Every frame is passed to two MediaPipe detectors running simultaneously.

02

Face Landmarks

MediaPipe returns 478 precise 3D points across your face every frame — eyes, brows, nose, lips, skull outline.

03

Gesture Detection

Eye Aspect Ratio math detects blinks and winks. Custom brow geometry catches eyebrow raises. A scikit-learn classifier identifies head direction.

04

Key Injection

Gestures are converted to keyboard inputs and sent directly to the game process using pyautogui and the Windows PostMessageW API.

Get Started

Up and running in under two minutes.

Requires Python 3.11, a webcam, and a microphone. No special hardware.

# 1. Create and activate virtual environment
python -m venv .venv
.venv\Scripts\activate

# 2. Install dependencies
pip install opencv-python mediapipe numpy pyautogui pygame pillow SpeechRecognition scikit-learn

# 3. Launch FacePlay
python launcher.py
